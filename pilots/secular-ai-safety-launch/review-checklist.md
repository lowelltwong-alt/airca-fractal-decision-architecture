# Review Checklist

## Decision review
- [ ] Is the launch question still correctly bounded?
- [ ] Are the release options complete enough for review?
- [ ] Is the recommended position explicitly named?

## Evidence and assertions
- [ ] Does each key claim have a source reference or review note?
- [ ] Are unsupported claims clearly marked as provisional?
- [ ] Are the highest-risk assertions visible to reviewers?

## Governance fit
- [ ] Does the selected path preserve human review where needed?
- [ ] Are rollback conditions defined?
- [ ] Are logging and traceability expectations explicit?

## AIRCA flow
- [ ] Architect layer defines guardrails
- [ ] Inform layer captures evidence and signals
- [ ] Rank layer explains recommendation
- [ ] Commit layer names accountable approval
- [ ] Act layer describes the rollout path
