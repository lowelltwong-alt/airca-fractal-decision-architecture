# Pilots

This folder contains applied pilots that prove the repository can move from concept architecture into structured, reviewable decision artifacts.

## Purpose

The pilots layer exists to answer a simple question:

Can AIRCA + fractal structure produce a bounded, machine-readable decision package that a real team could review?

## Pilot design rules

Each pilot should include:
- a bounded question
- one or more candidate views or positions
- explicit assessments
- sourced assertions
- a review surface
- a machine-readable graph example
- a node-level artifact record

## Current pilot sequence

### 1. Secular AI safety launch pilot
A domain-neutral governance case showing how the framework handles launch decisions, risk review, and control gating.

Future pilots can extend this pattern into:
- Christian / LAIRCA theological governance
- policy compliance and regulatory review
- hiring, operations, or institutional decision cases
