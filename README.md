# AIRCA + Fractal Decision Architecture

AIRCA is an AI-era adaptation of RACI for decision-making under conditions of AI-assisted overload.

This repository develops three connected ideas:

- AIRCA as decision architecture
- fractal taxonomy
- fractal ontology
- fractal decision architecture

## Core claim

As AI makes answers cheaper, the scarce resource becomes decision capacity.

## Why this exists

Traditional RACI helped clarify responsibility in environments where information moved slower, workflows were more linear, and automation was more limited.

AIRCA extends that logic for environments where AI can surface signals, generate options, rank possibilities, and help trigger execution.

## AIRCA

- **Architect**: define goals, guardrails, criteria, and thresholds
- **Inform**: surface signals, context, and candidate options
- **Rank**: compare, filter, and prioritize decision paths
- **Commit**: attach human ownership and accountability
- **Act**: execute through workflow, automation, or operational follow-through

## Fractal extension

The same governing logic should repeat across levels:

- enterprise
- team
- workflow
- document
- issue
- task

The goal is a stable structure where the same primitives, logic, and relationships recur across layers rather than breaking apart between them.
