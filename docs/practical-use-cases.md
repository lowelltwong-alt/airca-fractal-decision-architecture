# Practical AIRCA Use Cases

AIRCA becomes valuable when it helps people make better decisions with more accountability and less ambiguity.

## 1. Architecture decisions

Use AIRCA to record:

- the decision question
- the options considered
- the ranking criteria
- the final commitment
- the action plan

This gives teams a more structured alternative to loose architecture notes.

## 2. AI output review

Use AIRCA when a human needs to evaluate and approve AI-generated outputs.

Examples:

- policy drafts
- technical recommendations
- research summaries
- high-impact customer communications

## 3. Product prioritization

Use AIRCA to compare roadmap choices using explicit criteria and recorded rationale.

## 4. Governance and exception handling

Use AIRCA for approvals, exceptions, or policy tradeoffs where evidence and accountability matter.

## 5. Team-level recursive decisions

Use AIRCA at multiple levels:

- company
- function
- team
- workflow
- task

This expresses the fractal logic in a practical way.

## Best first use case for this repo

The best first use case is **AI-assisted architecture and governance decisions**.

Why:

- native fit with GitHub
- easy to demo
- easy to store in markdown and JSON
- useful for both humans and AI systems
