# Fractal Taxonomy Notes v1

## Working premise

Fractal taxonomy is a recursive classification logic that can repeat across levels without losing coherence.

The goal is not simply to create a larger hierarchy. The goal is to create a classification structure in which the same organizing logic can recur across enterprise, team, workflow, document, issue, task, and node levels.

## Why this matters

A normal taxonomy can help organize information, but it often becomes brittle as systems grow.

That brittleness shows up when:

- categories drift across levels
- similar objects are classified differently in different places
- inheritance becomes inconsistent
- AI retrieval becomes noisy
- future concepts do not fit cleanly into the structure
- governance rules attach unevenly because the object class is unclear

Fractal taxonomy is meant to reduce those problems by making classification more stable, recursive, and extensible.

## Core idea

The central claim is:

**the same classification logic should be able to recur across levels of the system.**

That does not mean every object is identical.  
It means every object can be classified through a familiar structural pattern.

For example, many nodes may still be describable through recurring fields such as:

- identity
- type
- role
- parent
- child
- status
- lifecycle
- related entities
- governing rules

This makes the structure more reusable and interpretable over time.

## Fractal taxonomy vs. ordinary taxonomy

### Ordinary taxonomy
Ordinary taxonomy is often:
- static
- hierarchical in a narrow sense
- useful for filing and navigation
- more brittle under growth

### Fractal taxonomy
Fractal taxonomy is:
- recursive
- reusable across levels
- better suited to inheritance
- stronger for machine-readable systems
- more adaptable under growth

## Relationship to ontology

Taxonomy helps answer:
- What is this?
- What category does it belong to?
- What kind of object is it?
- What level is it at?

Ontology helps answer:
- What does it relate to?
- What depends on it?
- What does it constrain or enable?
- What meaning does it carry in context?

Fractal taxonomy does not replace ontology.  
It gives ontology a more stable classification layer to work on.

## Relationship to decision-making

Fractal taxonomy also supports decision-making.

Classification helps determine:
- what kind of governance applies
- what review cycle may apply
- what escalation logic may apply
- what lifecycle treatment may apply
- what kinds of actions are valid or invalid

That means taxonomy is part of making a system not only organized, but decision-ready.

## Stable field families

One of the strongest expressions of fractal taxonomy is the use of stable field families that can recur across levels.

Examples include:

- identity
- classification
- relationships
- logic
- governance
- evidence
- provenance
- lifecycle
- inheritance
- interpretation rules

These are not just metadata buckets. They are part of the recursive classification logic that lets the same structure work across multiple scales.

## Inheritance and extensibility

A strong fractal taxonomy should support both inheritance and controlled extensibility.

That means:
- some values can flow downward
- some fields should never be inherited
- some child nodes may override parent values
- stricter local controls may be allowed
- future concepts should fit into the existing structure before new field groups are created

This reduces schema sprawl and keeps the taxonomy durable.

## Example across levels

The same taxonomy logic can recur across different scales.

### Enterprise level
A strategic initiative can be classified by type, role, owner, lifecycle, and related entities.

### Workflow level
A workflow object can be classified by process type, dependency class, review cycle, and operational role.

### Document level
A document artifact can be classified by artifact class, durability, state, and audience.

### Task level
A task node can be classified by action type, escalation category, lifecycle state, and governing rules.

The object changes, but the classification grammar remains recognizable.

## Working thesis

Fractal taxonomy is not just a better filing system.

It is a recursive classification layer that helps preserve structure, support ontology, and make decision systems more stable and scalable in AI-assisted environments.
