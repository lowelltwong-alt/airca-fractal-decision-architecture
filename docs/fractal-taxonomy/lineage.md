# Fractal Taxonomy Lineage

This note records the lineage, attribution boundaries, and conceptual sources for the fractal taxonomy node.

## Purpose

The purpose of this file is to distinguish among:

- external conceptual influences
- schema-derived structural logic
- repository-specific synthesis
- original extensions developed in this repository

This matters because the fractal taxonomy node draws on both broader ideas about classification and specific repository work on recursive structure and machine-readable artifacts.

## Conceptual lineage

The fractal taxonomy node is informed by broader ideas related to:

- classification systems
- hierarchical representation
- recursive structures
- multi-scale systems
- type systems
- machine-readable metadata design

This repository does not claim those broader domains as original. They are adjacent conceptual foundations that help explain why recursive classification matters.

## Schema-derived structure

The fractal taxonomy node also draws heavily on the fractal decision artifact schema and template work that already established:

- recursive nodes
- stable field families
- classification containers
- inheritance logic
- lifecycle layering
- interpretation rules
- structured extensibility

These schema materials provide much of the practical structural backbone for how fractal taxonomy is understood here.

## Repository-specific synthesis

Within this repository, fractal taxonomy is developed as more than a filing structure.

It is treated as:

- a recursive classification layer
- a stable grammar for object identity across levels
- a support for ontology
- a support for decision-making
- a support for machine-readable artifact design

That synthesis, in this form, is developed here by **Lowell T. Wong**.
