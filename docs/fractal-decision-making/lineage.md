# Fractal Decision-Making Lineage

This note records the lineage, attribution boundaries, and conceptual sources for the fractal decision-making node.

## Purpose

The purpose of this file is to distinguish among:

- external conceptual influences
- AIRCA-related lineage
- schema-derived structural logic
- repository-specific synthesis
- original extensions developed in this repository

This matters because the fractal decision-making node draws on both broader decision-theory and systems ideas, and specific repository work on AIRCA, machine-readable artifacts, recursive structure, and higher-order governance.

## Conceptual lineage

The fractal decision-making node is informed by broader ideas related to:

- decision intelligence
- Bayesian decision theory
- control systems and cybernetics
- constraint optimization
- operations research
- decision landscapes
- topology of trade-offs
- latent structure and local/global fit

This repository does not claim those broader domains as original. They are neighboring conceptual frameworks that help explain why recursive decision structure matters.

## AIRCA-related lineage

The fractal decision-making node also draws in part on AIRCA as a decision framework that emerged through a collaborative process between Alex A. and Lowell T. Wong.

Within this repository, AIRCA is extended into:

- recursive decision structure across levels
- machine-readable decision logic
- lifecycle-aware decision nodes
- persistent decision records
- UAIRCA and LAIRCA as higher-order extensions

This node therefore does not treat AIRCA as an isolated acronym. It treats it as part of a broader architecture for structured movement from signal to accountable action.

## Schema-derived structure

The fractal decision-making node also draws heavily on the fractal decision artifact schema and template work that already established:

- recursive nodes
- logic fields
- governance fields
- persistence fields
- lifecycle fields
- inheritance logic
- interpretation rules
- decision-ledger structures

These schema materials provide much of the practical structural backbone for how fractal decision-making is understood here.

## Repository-specific synthesis

Within this repository, fractal decision-making is developed as more than repeated approvals or repeated process checkpoints.

It is treated as:

- a recursive decision layer
- a stable decision grammar across levels
- a way of preserving accountability in AI-assisted systems
- a support for taxonomy and ontology
- a support for machine-readable artifact design
- a way of linking local choices to broader institutional coherence

That synthesis, in this form, is developed here by **Lowell T. Wong**.

## Higher-order extension

A distinctive extension in this repository is that fractal decision-making is also connected to:

- Ultimate Frame
- UAIRCA
- LAIRCA
- theological decision architecture
- moral anthropology
- Christian and non-Christian higher-order governance commitments

This means the node is not limited to procedural decision logic. It is also linked here to higher-order questions of legitimacy, authority, and the framing of decision criteria themselves.

## Attribution boundary

This node distinguishes among:

- broader external decision and systems concepts
- AIRCA as collaboratively emergent
- schema logic as a practical structural foundation
- repository-specific development of fractal decision-making and its higher-order extensions

## Working thesis

The fractal decision-making node is where recursive decision logic becomes part of a broader architecture for accountability, governance, higher-order commitments, and machine-readable institutional memory rather than remaining only a local decision method.
