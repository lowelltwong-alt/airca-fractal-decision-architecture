# First Doctrinal Cluster: Justification

This cluster is the first end-to-end doctrinal implementation of the AIRCA framework core.

It demonstrates how a doctrine can be modeled as:

- a topic
- multiple competing views
- tradition-scoped assessments
- typed dependencies and contradictions
- provenance-bearing assertions
- downstream coherence implications

## Why justification

Justification is a strong pilot domain because it is central, historically dense, highly interdependent, and contains major Protestant and Catholic divergences that can be modeled without flattening them.

## Core doctrine topic

- justification

## Included views

- forensic imputation
- infused righteousness
- synergistic covenantal participation
- moral influence / reductionist view
- antinomian distortion

## Modeling goal

This is not meant to declare universal truth claims inside the core engine.

Instead it demonstrates how the same doctrinal field can contain:
- affirmed views
- contested views
- rejected views
- false doctrines
- scoped baselines
- measurable coherence implications
