# Fractal Ontology Lineage

This note records the lineage, attribution boundaries, and conceptual sources for the fractal ontology node.

## Purpose

The purpose of this file is to distinguish among:

- external conceptual influences
- schema-derived structural logic
- repository-specific synthesis
- original extensions developed in this repository

This matters because the fractal ontology node draws on both broader ideas about relationships and meaning, and specific repository work on recursive structure, machine-readable artifacts, and multi-level decision architecture.

## Conceptual lineage

The fractal ontology node is informed by broader ideas related to:

- ontology and semantic structure
- graph theory
- knowledge graphs
- entity and relationship models
- dependency mapping
- semantic continuity across levels
- context-preserving representation

This repository does not claim those broader domains as original. They are neighboring conceptual foundations that help explain why recursive relationship logic matters.

## Schema-derived structure

The fractal ontology node also draws heavily on the fractal decision artifact schema and template work that already established:

- recursive nodes
- stable field families
- relationship containers
- provenance and evidence structures
- inheritance logic
- interpretation rules
- entity-model fields
- structured extensibility

These schema materials provide much of the practical structural backbone for how fractal ontology is understood here.

## Repository-specific synthesis

Within this repository, fractal ontology is developed as more than a relationship map.

It is treated as:

- a recursive meaning layer
- a way of preserving semantic continuity across levels
- a support for taxonomy
- a support for decision-making
- a support for machine-readable artifact design
- a way of keeping local objects legible in broader system context

That synthesis, in this form, is developed here by **Lowell T. Wong**.

## Attribution boundary

This node distinguishes among:

- broader external ideas about ontology, semantics, and graph structure
- schema logic that already provides a practical structural foundation
- repository-specific development of fractal ontology as part of a larger decision architecture

## Working thesis

The fractal ontology node is where recursive relationship logic becomes part of a broader architecture for meaning, governance, and decision-making rather than remaining only an abstract semantic layer.
