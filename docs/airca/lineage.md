# AIRCA Lineage

This note records the lineage, collaborative emergence, and attribution context for AIRCA within this repository.

## Purpose

The purpose of this file is to distinguish among:

- collaborative emergence
- intellectual lineage
- repository-specific extensions
- authorship and attribution boundaries

This matters because AIRCA did not emerge here as an isolated idea detached from process, artifact structure, or dialogue.

## Collaborative emergence

The AIRCA framework emerged through a collaborative process between Alex A. and Lowell Wong.

It took shape through:

- an artifact created by Lowell Wong
- Alex A.’s interpretation and interrogation of that artifact through AI
- the interaction between the document’s structure, research content, and linked JSON, which helped surface and connect the ideas across the artifact

This repository therefore does not present AIRCA as having appeared in isolation or as a purely abstract naming exercise detached from artifact design.

## AIRCA and RACI

Within this repository, AIRCA is treated as an AI-era adaptation of RACI for decision-making under AI-assisted overload.

That means AIRCA is not being framed merely as a generic acronym or a replacement management slogan. It is being treated as a decision architecture that becomes necessary when:

- AI can surface signals continuously
- options can be generated at scale
- ranking and narrowing can happen before execution
- automation can execute actions faster than institutions can always govern them informally

In that sense, AIRCA is understood here as a development beyond traditional role-clarity models toward a fuller decision-flow model.

## Repository-specific extension

This repository’s specific extension of AIRCA is into:

- fractal taxonomy
- fractal ontology
- fractal decision-making
- machine-readable artifact design
- recursive governance structures

That extension is primarily the work developed here by Lowell Wong.

## Attribution boundary

This repository distinguishes between:

- AIRCA as a collaboratively emergent framework
- broader external intellectual influences
- the fractal and machine-readable extensions developed in this repository

This means the repository aims to give credit without collapsing all contributions into a single undifferentiated claim.

## Working thesis

AIRCA is best understood here not as an isolated framework, but as part of a broader architecture in which decision logic, recursive structure, and machine-readable artifacts reinforce one another.
