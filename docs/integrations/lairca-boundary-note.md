# LAIRCA Boundary Note

## Naming clarification

Within the broader theological project, **LAIRCA** is the Christian adaptation layer of AIRCA.

- **AIRCA** remains the secular business and governance decision architecture core.
- **LAIRCA** is the Christian adaptation layer.
- **Logos** is the broader theological project within which LAIRCA is only one part.

## Repository boundary

This repository should remain primarily focused on:

- secular business decision architecture
- governance and accountability design
- organizational ontology and taxonomy
- machine-readable artifacts for workflow and decision systems
- cross-domain AIRCA implementation

Deep doctrinal and theological implementation should live in the separate Logos repository:

- `lowelltwong-alt/logos-fractal-theological-architecture`

Within that repo, AIRCA-derived Christian decision architecture should be developed under the **LAIRCA** layer or node.

## Practical rule

Use AIRCA here as the reusable core.

Use LAIRCA in the Logos repo when the work becomes explicitly theological, doctrinal, ecclesial, or Logos-grounded.
