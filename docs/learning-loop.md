# AIRCA Learning Loop

AIRCA should not stop at documentation.

It should learn from repeated decisions over time.

## Learning questions

After a decision is made, AIRCA should ask:

- did the outcome match the expectation?
- were the criteria useful?
- where did human reviewers override the default choice?
- what evidence was missing?
- where was cognitive load highest?
- should similar decisions be escalated differently next time?

## Minimal implementation

The first AIRCA learning loop can store:

- review horizon
- outcome summary
- signals to watch
- reviewer notes
- overrides

## Why this matters

This is what turns AIRCA into:

- a self-learning decision system
- a white-paper-generating system
- a durable organizational memory layer
