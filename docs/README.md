# Docs

This folder contains the substantive concept notes and concept nodes for the repository.

## Purpose

The `docs/` layer is where the repository moves from top-level framing into deeper concept development.

It is also where the repository begins to take on a more fractal structure, meaning that major concepts become their own nodes with their own internal logic, notes, and future machine-readable metadata.

## Current structure

### Core notes
These documents develop the repository’s major ideas directly:

- `airca-notes-v1.md`
- `fractal-taxonomy-ontology-decision-model.md`
- `technical-and-conceptual-lineage.md`

### Concept nodes
These folders are the beginning of the more fractal structure:

- `airca/`
- `fractal-taxonomy/`
- `fractal-ontology/`
- `fractal-decision-making/`
- `integrations/`

## What the concept nodes mean

### `airca/`
The AIRCA node develops AIRCA as an AI-era adaptation of RACI and as a decision spine for AI-assisted environments.

### `fractal-taxonomy/`
The fractal taxonomy node develops recursive classification logic and stable field families that can recur across levels.

### `fractal-ontology/`
The fractal ontology node develops recursive relationship logic and semantic continuity across levels.

### `fractal-decision-making/`
The fractal decision-making node develops recursive decision structure, decision rights, review logic, and AIRCA as a scalable decision spine.

### `integrations/`
The integrations node develops how the major concepts connect as an architecture, including machine-readable artifacts, artifact naming, anchors, tags, and crosswalks.

## Why this structure matters

The repository is not only collecting notes. It is beginning to model the same idea it argues for:

- stable structure across levels
- recursive concept nodes
- human-readable entry points
- machine-readable architecture beneath them

That is what makes the `docs/` layer the first real fractal content layer of the repository.

## Likely next evolution

Over time, each concept node may include its own:

- `README.md`
- `artifact.json`
- lineage note
- substantive notes
- examples
- machine-readable schemas or objects

As that happens, the repository will move closer to a fully fractal internal architecture.
