# Decision Paper: Choose AIRCA MVP

## Abstract
This paper documents a structured AIRCA decision and turns it into a reusable public-facing record.

## Problem
What should AIRCA ship first for public adoption?

## Context
The project needs a useful first release that people can understand and try quickly.

## Options Considered
### Option 1: Schema + CLI
Ship a schema, CLI, and example artifacts first.
- Total score: 14
- Evidence: Fastest useful MVP, Easy to demonstrate

### Option 2: Full orchestration engine
Build a larger workflow engine before examples and packaging.
- Total score: 5
- Evidence: Powerful later, Too heavy for first release

## Evaluation Criteria
- adoption
- clarity
- difficulty

## Decision
Schema + CLI

## Rationale
Schema + CLI creates the fastest path to usability, demos, and GitHub-native adoption.

## Implications
- Stabilize schema
- Publish examples
- Add GitHub validation
